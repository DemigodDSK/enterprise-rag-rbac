# SOP: Change Management
Company: RT Healthcare
Owner: Engineering & Compliance
Version: 2.0

---

# 1. Purpose

To ensure all system changes are reviewed, approved, tested, and documented to reduce risk.

---

# 2. Change Categories

- Standard Change
- Emergency Change
- Major Change

---

# 3. Change Request Requirements

All changes must include:

- Description
- Business justification
- Risk assessment
- Rollback plan
- Testing evidence
- Impact analysis

---

# 4. Approval Workflow

- Technical review
- Security review (if applicable)
- Manager approval
- CAB approval (for major changes)

---

# 5. Implementation

Changes must be:

- Deployed via CI/CD pipeline
- Logged
- Monitored post-deployment

---

# 6. Emergency Changes

- Must be documented within 24 hours
- Post-implementation review required
